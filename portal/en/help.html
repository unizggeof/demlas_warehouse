Overview

DEMLAS warehouse portal serves a gateway to the project information infrastructure. Any user can use it to search for available resources of heterogeneous types as geospatial datasets, maps, observations as well as accompanying non-spatial documents, sheets, field notes, etc. Simple full text searching capability is offered with matching keywords autocomplete functionality. In addition thematic and geo searches options are available.

How to access it?

Discovery Map portal application is available from the project web site.

How to use it?

Full text search

Use the search to write a full text query, e.g. Dodaci popisu and launch search by clicking on or hitting enter (return) key.

Thematic search

By choosing the parent data theme you can restrict the result list, e.g. if you wanna find the data for orthoimagery, select Orthoimagery option from the menu.

Geospatial search

Three options are provided:

  1. Spatial Extent by activating (ON) only the data having boundaries within the current screen extent will be searched.
  2. Overlapping POI by activating (ON) only the data having boundaries overlaped the point identified by click on map will be returned.
  3. Current Geolocation by activating (ON) only the data having boundaries overlaped the current device geolocation will be returned.

Only one options is allowed at the same time and the one selected will be indicated by the red bar at the bottom of the screen

Buttons

Four buttons are provided:

  1. Geolocation will geolocate the device on map and provide basic metadata resulting from the location solution.
  2. Apps will open modal from where user can navigate to applications developed and deployed for the warehouse infrastructure. Individual warehouse applications are described later in this help. The footer provides link to the project web page.
  3. Layers will open simple list of map layers grouped into the parent data themes and Base maps.
  4. Help will open this page.

Overview

Manager app offers web based graphic user interface serving project stakeholders (researchers, students and externals) a tool to share and document spatial and non-spatial data as files. It has been developed as a responsive application and thus you can use it on various desktop, laptop and mobile platform devices.

How to access it?

The application is accessible from the Apps modal window available from the warehouse initial portal. It is accessible only for authorized users. In order to get access to the manager you have to contact the system admin and request user and password. Testing users are also made available upon request.

How to use it?

After a succesfull login, you will see the listing of top folders representing the basic data themes: e.g. cp is meant to manage data related to cadaster parcels, lc to land cover, elu to existing land use data, etc. This level can be updated only by the system admin. Therefore, if you find not any theme corresponding your data, e.g. hydrology, or oceanography, please contact the system admin.

Once you enter into a sub-directory, options to create directory and upload a file are provided.

Once you enter a directory containing files, you will see basic metadata as Name, Size and Modification date for each file. In addition you will find buttons as Create / Update metadata using demlas metadata editor and Delete which will delete the data file permanently. In addition, those geospatial data files currently supported by the warehouse (Geospatial TIF) are labeled as Published if they have already been published in the warehouse geoserver.

Sample user scenario

Lets imagine a field work where we collect apart of real surveyoing data also sort of supporting data as images, scatches, notes etc. We can do it directly into digital forms or first write / draw on a paper. You can use the manager app to store this data in warehouse immediatelly from the field, thus make them accessible to other colleagues.

In addition, you may also directly upload the measurement raw data from controllers of the geodetic instruments connected to the Internet and immediatelly share with other colleagues at other stations or in office.

Overview

Metadata editor is a light web application offering users a web form to create documentation of data or any other internal / external information resources.

How to access it?

The application is accessible from the Apps modal window available from the warehouse initial portal. Editor is currently available with no user authentication required and thus anyone who needs to create and download metadata to a local machine can use it. For internal stakeholders authentication will be required in order to allow local storage and publishing in catalogue functionality.

How to use it?

Basic set of metadata fields providing information such as What? are you gonna describe containing the data title, description and keywords if any applicable. Later information on Who? providing basic contact details as full name and email address. In order to determine the geographic location information on Where? is to be provided in form of map by panning and zooming on area of interest. Further information on How? the data was created and can be accessed by others is described, and finally temporal When? information on data creation, modification, publication or other relevant is to be input here.

The editor offers more metadata elements to be filled by data provider in case he/she feels a need to do so. This section can be opened/closed by button

Once the metadata has been filled, you can use the button to save XML file encoded using Dublin Core metadata schema on your computer. In order to save the metadata in the warehouse use button and if you wanna also publish metadata in warehouse CSW catalogue use button

Overview

Geoserver is an open soruce Web GIS server application and serves for geospatial data provision in warehouse. It basically conencts to supported raster / vector data encodings and publishes them using standard geospatial web services as Web Map Service (WMS)for data visualisation in a form of queryable web maps, Web Feature Service (WFS) for vector data and Web Coverage Service for raster data querying and downloading. Each dataset published via warehouse manager and recognized as geospatial is exposed on Web using Geoserver API. Currently the warehouse supports publishing of tif raster data as Tagged Image File Format with geographic information (Geoserver GeoTIFF raster store type)and TIFF raster data accompanied by a spatial data file and projection file (Geoserver WorldImage raster store type). Geoserver provides its own administration interface which can be used to publish further supported data types (e.g. ShapeFiles, Postgis database). Even more raster and vector data types are supported by additional modules. In order to access the Geoserver admin Web GUI contact the system admin for further instructions.

Web services for data provision

If you would like to use the geospatial data available in warehouse from remote GIS applications e.g. QGIS or ArcGIS you can use the following OGC services:

Web Map Services (WMS)

WMS is the mostly used web service operating on both raster and vector geospatial data and generates map preview with a dynamic way of layer style symbolism each associated with a legend. 

DEMLAS Web Map Service (WMS) for Cadastral Parcel Data Theme

DEMLAS Web Map Service (WMS) for Orthoimagery Data Theme

DEMLAS Web Map Service (WMS) for Land Cover Data Theme

Web Feature Services (WFS)

WFS is a web service operating on vector geospatial data and provides query and transactions mechanism to access and retrieve geometric and attribute data at the feature level in real time on the Web. 

DEMLAS Web Feature Service (WFS) for Cadastral Parcel Data Theme

DEMLAS Web Feature Service (WFS) for Orthoimagery Data Theme

DEMLAS Web Feature Service (WFS) for Land Cover Data Theme

Web Coverage Services (WCS)

WCS is a web service for accessing and processing of raster data, more generally: coverages of multidimensional gridded data, extending WMS by formats used for complex modelling and analysis, usually used to encode large datasets for meteorology, oceanography and climatology. 

DEMLAS Web Coverage Service (WCS) for Cadastral Parcel Data Theme

DEMLAS Web Coverage Service (WCS) for Orthoimagery Data Theme

DEMLAS Web Coverage Service (WCS) for Land Cover Data Theme

Overview

Catalogue component is serving warehouse as data documentation management tool. Metadata can be stored, indexed, searched and shared using standardized interfaces. The catalogue is connected to the warehouse portal and used to search for data via metadata. Current warehouse catalogue implementation is based on Geonetwork open source catalog application. By using the warehouse catalogue CSW service endpoint, you will be able to search and use the data in remote applications, e.g. QGIS or ArcGIS, etc. In order to access the geonetwork admin Web GUI contact the system admin for further instructions.
